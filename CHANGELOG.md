# Changelog

## 0.3.2 - 2026-06-26

- 修复：余额抓取加入来源优先级；已有成功余额标签缓存的站点优先使用 DOM 标签锚定余额，避免维云、Aicard、paidyun 等站点被泛化 API 递归扫描误抓额度/卡密/配额数字。
- 修复：DOM 余额提取优先在标签所在小容器内取金额，支持 `¥ / ￥ / $` 金额符号，避免扫到后续充值档位；同时不再强化 `余额`、`额度` 这类泛词缓存，防止错误自学习污染。
- 修复：DOM 余额命中后改为同一路径多次采样再确认，避免 SPA 初始渲染阶段把充值档位误当余额。
- 增强：新站点即使 API 已抓到余额，也会先做一次轻量 DOM 复核；无缓存标签时只探优先路径并限制采样次数，提升通用性同时避免明显拖慢检测。
- 保持：未知站点仍保留 API 余额优先，分组与倍率仍优先走 API，避免影响现有分组展示稳定性。

## 0.3.1 - 2026-06-26

- 修复：补齐 health_checker.py 缺失的 sys 导入，避免后端导入时因 `_MEIPASS` 路径判断崩溃，导致 EXE 窗口已打开但 8084 服务未启动。
- 修复：打包脚本现在会先安装 backend/requirements.txt，避免 EXE 缺失 FastAPI / Uvicorn / Playwright 运行依赖。
- 修复：PyInstaller spec 显式加入 FastAPI、Uvicorn、Starlette、Pydantic、Playwright 的 hiddenimports，避免窗口模式下后端线程静默起不来。
- 修复：打包入口改为直接使用现成 spec，减少命令行参数分叉导致的漏包风险。

## 0.3.0 - 2026-06-25

- 重写：DOM 余额提取算法 v5。彻底放弃全文正则猜数字的策略，改为遍历 DOM 文本节点 + 标签优先级字典 + 上下文窗口扫描。
- 自学习：probe_cache.json 现在不仅记成功路径，还记成功标签。下次刷新时，缓存标签得 +1000 优先级，几乎不会错位。
- 标签字典扩充：覆盖"我的资产/总资产/金币池/积分余额/点数/Available/Remaining"等更多花式说法。
- 探测路径扩充：新增 /user/asset-source、/user/dashboard、/user/wallet、/user/profile、/account、/account/billing、/billing。
- 营销词过滤改进：只检查标签到数字之间的内容，避免误伤数字之后无关的"消费"、"送礼"等字眼。
- 登录态检测加强：中英文关键词分开匹配，英文统一小写比较，覆盖 Dashboard / API Keys / Billing / Profile / Usage 等。
- 出厂预设：probe_cache.json 打包进 dist。新机器首次启动自动 seed 14 个常见站点的路径与标签，零冷启动。
- 截图保护：仅在已抓到分组（说明已登录）但余额仍为 None 时才保存调试截图，避免离线/未登录站点的无意义快照堆积。

## 0.2.9 - 2026-06-25

- 修复：DOM 余额提取增加重试机制（最多 3 次），解决 SPA 页面余额内容加载延迟导致抓取为空的问题。
- 修复：只保留包含余额关键词的页面文本作为候选，避免无余额页面的文本干扰提取。

## 0.2.8 - 2026-06-25

- 修复：DOM 余额提取增加营销文案过滤（满/送/减/折/优惠等），避免抓到促销数字作为假余额。
- 修复：DOM 余额提取增加长度过滤，超过30字符的片段视为营销文案跳过。
- 修复：多个候选余额值时取最小值（真实余额通常是最小的数字）。
- 新增：余额关键词增加"额度"、"信用余额"、"可用积分"等。

## 0.2.7 - 2026-06-25

- 修复：DOM 余额提取增加"已登录"验证，页面无控制台/钱包等登录态关键词时不再提取余额，避免抓到营销文案或价格展示的假数据。
- 新增：API 端点探测增加 `/api/user/balance`、`/api/billing/info`、`/api/me`，减少对 DOM 提取的依赖。

## 0.2.6 - 2026-06-25

- 修复：启动时窗口自动居中显示。

## 0.2.5 - 2026-06-25

- 修复：API 返回全部非数值倍率（如"自动"）时，不再设空 groups 覆盖 DOM 兜底，确保分组信息可从页面正确回退提取。

## 0.2.4 - 2026-06-25

- 修复：分组倍率提取时，若 API 返回非数值字符串（如 "自动"），跳过该项而非崩溃或塞入错误的 1.0x 默认值。

- 修复：分组倍率提取时，若 API 返回非数值字符串（如 "自动"），跳过该项而非崩溃或塞入错误的 1.0x 默认值。
- 修复：`group_ratio` / `usable_group` 增加 `isinstance(dict)` 类型守卫，防止 API 返回 list 时 `.items()` 崩溃。
- 修复：Playwright `page` 变量提前初始化为 `None`，`finally` 块中统一做 None 检查，防止 `new_page()` 失败后 `NameError`。
- 影响范围：New API group_ratio、New API user/groups、Sub2API rate_multiplier 三处提取路径均已加固。

## 0.2.0 - 2026-06-21

- 改为父子卡片展示：站点层显示余额和整体状态，分组层显示倍率和价格。
- 状态判定改为保守状态机：online / unknown / error，不乱判离线。
- 探明 New API 系列站点的 `/api/pricing` 接口结构（含 group_ratio、usable_group）。
- 探明当前 ccswitch key 对余额接口返回未授权，确认余额需通过网页登录态抓取。
- 确认下一步方向：分组/倍率/余额全部改为网页抓取。
- 新增 HANDOFF.md 项目交接文档。
